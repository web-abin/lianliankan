## Why

微信小游戏需要一份可执行的**产品与需求基线**，把连连看核心玩法、经济闭环、道具获取规则、成就与主题、云端同步与排行榜等已定稿结论固化下来，避免开发与验收口径漂移。本变更仅产出 **OpenSpec 文档**（proposal / design / specs / tasks），不直接改业务代码。

## What Changes

- 新增「连连看」产品的 **需求规格**：核心规则、普通闯关与每日挑战、血量与失败、放弃挽回弹窗、铺砖与 **关卡机制**（单方向重力、迷雾、翻牌；无双重力）、**主线 JSON（平台期前）+ 之后沿用末关难度**、**开局铺砖自下而上每行 bounce**、**有特殊机制的关卡：铺砖后 2 秒蒙层 + 关卡旁图标 + 中央 Toast 复查**、局内 **选中 bounce / 连线光束 / 消除星光与飞入宝箱**、**全局共用 IP 异形弹窗壳 + bounce 入场**，以及特效与音效要求。
- 新增 **经济与商店**：金币来源与签到表、商店单价、道具仅在局内通过分享获得（每次分享 +3 个）、道具在主页商店用金币预购、50 金币买 1 滴血等。
- 新增 **成就与主题**：主题仅通过成就解锁；成就体系预留未来发金币/音效等非主题奖励。
- 新增 **数据与排行**：每日挑战仅存本地；用户资料与进度等存 Supabase，经 Edge Function 登录；世界榜排序规则与步数字段定义。
- 新增 **微信侧**：圈子/桌面/喊人/客服等能力在规格中的边界说明；好友榜与世界榜数据源区分。
- 无 **BREAKING** 代码契约（本仓库尚无对应实现 spec 的代码变更）。

## Capabilities

### New Capabilities

- `llk-gameplay-core`：连连看路径规则、普通闯关与每日挑战模式、血量与耗尽行为、放弃挑战挽回弹窗、无通关成功弹窗与进关铺砖（**自下而上每行 bounce**）、关卡机制（单方向重力四向阶梯填补、迷雾、翻牌；**无双重力**）、**有特殊机制关卡：铺砖后蒙层约 2 秒 + 关卡旁图标 + Toast 复查**、**局内选中 bounce / 连线光束 / 消除星光与飞入宝箱**、**主线 JSON 配置（仅 1…L 关；n>L 沿用第 L 关难度仅换种子/布局）**、**每日挑战独立生成器（固定最大棋盘、每日双维随机组合、无双重力）**、局内特效与音效、主题全局影响牌面。
- `llk-economy-store`：金币获得与连续签到表、商店定价、道具分享与预购规则、买血与喊人回血。
- `llk-achievements-themes`：成就系统、主题仅成就解锁、三档自定义主题与默认美食主题、后续扩展非主题奖励。
- `llk-sync-rank-supabase`：Supabase + Edge Function 登录、本地与云端字段划分、世界排行榜排序（关卡数、最高关步数、通关时间戳）及与旧「创建时间」表述的关系。
- `llk-wechat-shell`：首页模块（圈子、喊人、桌面、签到、挑战、商店、主题、排行）、设置与客服、分享与后续激励视频占位、**全局共用弹窗壳（IP、异形底板、bounce）**、性能分包与非功能要求。

### Modified Capabilities

- （无）`openspec/specs/` 下尚无既有能力文档。

## Impact

- **文档**：`openspec/changes/link-game-product-spec/` 下 `proposal.md`、`design.md`、`specs/**/spec.md`、`tasks.md`。
- **实现**：后续 `/opsx:apply` 将按 `tasks.md` 驱动代码；当前变更不修改 `src/`。
- **外部系统**：Supabase（数据库、Edge Functions）、微信开放平台能力（登录、分享、客服、开放数据域等）。
