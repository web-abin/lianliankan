## Context

- 项目为 **微信小游戏**，渲染栈为 **Pixi.js 6 + Matter.js + Vite**（见仓库 `AGENTS.md`）：无 DOM、需 adapter、触摸坐标直传、音频用 `wx.createInnerAudioContext` 等。
- 本变更将 **产品规则** 固化为 spec；实现阶段在 `src/` 中落地场景路由、商店、成就与云端同步。
- **Supabase**：用户昵称头像、金币、已购、关卡进度等经 **Edge Function** 完成登录与鉴权；密钥仅在 **`.env`** / 构建注入，**不得**打进客户端明文。
- **每日挑战**：仅存 **本地**（随机种子、今日完成状态等），与云端主线进度解耦。

## Goals / Non-Goals

**Goals:**

- 为玩法、经济、成就、同步与微信壳层提供 **一致的可验收口径**。
- 明确 **排行榜步数** 仅指「当前最高关卡那一局」的步数，与世界榜排序链一致。
- 明确 **道具** 与经济规则（见 `llk-economy-store`）；**主页商店** 用金币预购；与后续激励视频切换预留接口位。

**Non-Goals:**

- 本设计 **不包含** 具体 Pixi 节点层级、Shader 细节与 Matter 刚体参数（实现任务中再定）。
- **不包含** Supabase 表结构的最终 SQL（任务阶段可单列 migration）；此处只约定字段语义与同步方向。

## Decisions

1. **权威排序（世界榜）**  
   - **主键**：已通过关卡数 **降序**。  
   - **次键**：当前最高关卡 **该局** 步数 **升序**（步数少者优先）。  
   - **第三键**：达到 **当前最高关卡数** 时的 **通关时间戳**（物理时间，Unix 毫秒）；**更早**者优先（数值更小排名更前）。  
   - **废弃** 文档旧稿中「步数相同则创建时间倒排」的表述，以第三键时间戳为准，避免与「通关时间」语义冲突。

2. **血量为 0**  
   - **禁止** 开始普通闯关；**允许** 进入每日挑战、商店、首页其它入口。  
   - 回血：**50 金币 = 1 滴血**；**喊人 +3 血**（次数上限若需防刷，在实现/运营层配置）。

3. **道具来源双轨**  
   - **局内**：仅能通过 **分享** 增加库存（详见 `llk-economy-store`）。  
   - **局外（主页商店）**：**300 金币/类道具** 预购，带入局内库存。

4. **主题与成就**  
   - 主题 **仅** 成就解锁；成就模型 **可扩展** 为金币/音效等非主题奖励。  
   - 三个非默认主题与成就条件的具体对应见 `llk-achievements-themes` spec。

5. **数据分区**  
   - **本地**：每日挑战状态、客户端缓存、未同步队列（若有）。  
   - **Supabase**：openid 映射、展示信息、经济、主线关卡进度、排行榜用聚合字段。

6. **关卡机制三类**  
   - **重力掉落（阶梯填补）**：消除后按关卡设定方向（左/右/上/下）做类俄罗斯方块式填补，改变邻接关系。  
   - **迷雾**：图案被迷雾遮挡；点击或消除邻近区域可 **暂时** 驱散迷雾露出图案。  
   - **隐藏翻牌**：初始背面朝上，点击翻开；未成功配对的候选砖块在选中结束后 **自动翻回背面**。

7. **主线 JSON 与平台期**  
   - **平台期之前** 的每一关 SHALL 由 **JSON 配置** 描述（棋盘、种类数、重力单方向、迷雾、翻牌等）。  
   - JSON **仅包含第 1…L 关**（L = 条数）；**第 L 关** 即难度 **封顶模板**。  
   - 关卡序号 **n > L** 时，系统 SHALL **不再** 读取更高难度的配置；SHALL **始终沿用第 L 条 JSON 的难度参数**（硬指标与第 L 关一致），仅通过 **布局种子 / 随机摆放** 换花样。  
   - **不包含**「双重力」机制（单局内不组合两种重力规则）。

8. **每日挑战生成器（独立于主线）**  
   - 每日挑战 SHALL **不** 与主线共用同一套关卡/生成器逻辑。  
   - 棋盘尺寸 SHALL **固定为允许的最大规格**（与主线 JSON 第 L 关所定义的最大棋盘一致，或产品单独规定的最大格，二者应对齐）。  
   - **每个自然日** SHALL 从难度维度集合中 **随机选取两种** 进行 **组合** 生成当日挑战（例：迷雾+重力、翻牌+重力、高种类数+重力）；维度至少覆盖：**迷雾**、**单方向重力**、**翻牌**、**图案种类数**（种类多）。**不包含双重力。**  
   - 随机 SHALL 使用「日」级种子（如 `YYYYMMDD` + `userId` 可选）以保证同日可复现、次日刷新。

9. **全局弹窗与局内动效（产品一致性与验收口径）**  
   - 全站弹窗 SHALL 复用 **带 IP、异形底板、bounce 入场** 的共用壳（见 `llk-wechat-shell`）。  
   - 铺砖 SHALL **自下而上**逐行/层，**每行 bounce**；有特殊机制的关卡在铺砖结束后 **2 秒自动消失蒙层** + **关卡号旁图标** + 点击 **中央 Toast** 复查说明（见 `llk-gameplay-core`）。  
   - 局内反馈 SHALL 包含：**双砖橙框 + bounce/抖动**、合法连线的 **亮蓝半透明流光光束**、消除的 **爆裂空心环 + star.png 星粒（含重力）+ 「好！」飘字**、路径中点 **star2** 弧形飞入 **进度条/宝箱** 并触发 **进度条脉冲**；砖块为 **三层叠加**（深褐偏移投影 + 米白圆角卡面 + 顶缘高光 + 食物图标）；棋盘 **窄左右边距**、**横缝极小纵缝略大**；选中/消除音效 **先选中音结束再消除音**（详见 `llk-gameplay-core` 对应 Requirement）。

## Risks / Trade-offs

- **[Risk] 排行榜作弊** → 服务端校验通关上报、限频、异常步数检测；MVP 可先做「客户端上报 + 服务端存证」再加强。  
- **[Risk] 分享能力政策变更** → 文案与入口避免纯诱导；预留激励视频与分享并行配置。  
- **[Risk] 分包与主题图集体积** → 首包仅默认美食主题；其余主题按需下载（见 `llk-wechat-shell`）。  
- **[Trade-off] 每日挑战无榜** → 减少服务端复杂度；炫耀依赖分享卡片或截图。

## Migration Plan

- 本变更 **仅新增文档**；无数据库迁移、无线上回滚。  
- 代码落地时：先 **单机玩法 + 本地存** → 再 **接 Supabase 登录与同步** → 再 **开放数据域好友榜**。

## Open Questions

- `docs/prompt.md` 中「通关后」一句未完稿，是否在首版要求 **通关结算动画/飘字**；当前产品结论为 **无成功弹窗、直接进下一关**，与「通关后」补充内容需产品确认。  
- **喊人** 每日次数上限、是否冷却，由运营定数后写入 economy spec 的配置段。  
- 若同一关卡 **多种特殊机制并存**（例如重力+迷雾），开局蒙层与图标是 **合并一段说明** 还是 **分步蒙层**，可由实现阶段与美术稿统一；当前 spec 允许 **一条蒙层内合并说明** 以满足 2 秒自动消失节奏。
