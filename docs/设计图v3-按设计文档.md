# 设计图 v3 - 按设计文档生成 AI 生图提示词

本文档基于 `docs/设计文档.md` 整理，目标是把每个页面的功能点、布局结构和统一视觉语言，转换成可直接交给 AI 生成界面设计图的提示词。

使用方式：
- 先复制上方对应模型的 Prompt 模版。
- 再把下方某个页面的“具体设计提示词”替换进模版里的 `[页面内容]`。
- 这样就能快速拼接出一个完整、统一风格的页面生图 Prompt。

```text
- preserve clear hierarchy: primary action first, status second, decoration last
- all important text and icons must stay readable on busy backgrounds
- avoid dark themes, realistic rendering, glossy 3D, hard vector style, cyber UI, overcrowded layouts, tiny buttons, or sharp corners

```

## 豆包 Prompt 模版

```text
请生成一张微信小游戏竖屏界面设计图，整体必须保持统一的“治愈系儿童绘本插画风”，并且和参考图属于同一个视觉世界。

统一风格要求：
- 彩铅与水彩结合的质感，能看到纸张颗粒感、彩铅排线和轻微水彩晕染
- 线条是手绘感描边，带轻微抖动和粗细变化，描边颜色偏深墨绿色，不要机械矢量直线
- 整体气质温暖、明亮、松弛、童趣，像春日草地上的儿童绘本插画
- 固定视觉基调：浅蓝天空、嫩黄绿色草地、暖棕色卡皮巴拉、米白色卡片、暖橙色主按钮
- 所有页面必须保持同一套角色比例、同一套笔触语言、同一套按钮与卡片设计体系

画面与布局要求：
- 这是微信小游戏 UI 截图，不是纯插画海报
- 竖屏构图，全屏页按 750x1750 的长屏感组织画面
- 顶部信息必须避开状态栏和右上角微信胶囊按钮
- 底部主按钮必须避开底部手势安全区
- 主按钮始终是最强视觉焦点，辅助信息弱化但清楚
- 文字、图标、按钮边界都要清晰，不能糊，不能太小
- 弹窗必须居中，内部元素相对弹窗排布，四周留白舒适

负向限制：
- 不要写实摄影
- 不要暗黑压迫感
- 不要科技感 HUD
- 不要玻璃拟态和金属质感
- 不要硬朗锐角
- 不要拥挤排版
- 不要过度商业 Banner 感

页面内容：
[页面内容]
```

## 页面具体设计提示词

### 1. 加载页

```text
微信小游戏加载页。全屏背景改为日式露天温泉泡澡场景，延续治愈儿童绘本插画风。顶部是可裁切的浅蓝天空、树枝、蒸汽和柔和光斑留白区，不放核心信息。中部主视觉是一只泡在木质温泉池里的卡皮巴拉，只露出头部和上半身，表情非常放松，旁边可以有 1 到 2 只温和的小陪伴动物，例如小鸭或白团雀，但不能抢主角。背景有石头围边、远景树影、竹林或木屋一角，整体像清晨或傍晚的温暖温泉时光。中下区域放横向居中的加载进度条，宽度大约占屏幕 60% 到 72%，不要做科技感硬进度条，而是把容器设计成扁圆的木牌或温泉池口，内部用半透明白雾、暖色蒸汽和细小气泡从左到右缓慢填满。进度条上方有简短文案“正在准备今天的温泉时光”或“Loading...”。底部贴近安全区上方放健康游戏忠告，视觉权重最低但清晰可读。层级关系是：温泉卡皮巴拉最醒目，加载进度第二，文字提示第三，底部忠告最弱。
```

### 2. 首页

```text
微信小游戏首页，全屏竖屏，参考图同款治愈系儿童绘本草地世界。背景是固定首页场景，不随主题切换：浅蓝天空、远景树影、淡绿色草坡、蒲公英、白色小花、零星小动物，整体轻松、明亮、软糯。顶部状态区位于左上安全区内，横向排列设置按钮、金币显示、血量显示，整体放在浅米白半透明小胶囊卡片上，描边为深墨绿色手绘线条，不进入右上角胶囊危险区。中央展示区是页面视觉锚点，上方放游戏主标题“开心点连连看”，下方是大尺寸卡皮巴拉待机形象，角色完整、可爱、具有陪伴感。左右两侧围绕中央角色放两列竖向功能入口：左侧从上到下是圈子好礼、添加桌面、商店、主题；右侧从上到下是喊人、每日奖励、排行榜、每日挑战。每个入口都做成独立的小插画图标，不要科技按钮感，图标本身像木牌、日历、奖杯、星星、电话、礼物一类可爱手绘物件，图标下沿贴着中文标签文字，标签底边对齐。底部主行动区放一个全页最强视觉的暖橙色“开始游戏”大按钮，按钮宽厚圆润，位于底部安全区上方；按钮上方或右上角挂一个小木牌显示当前主线关卡，例如“第 4 关”。整体信息层级：开始游戏第一，金币和血量第二，外围功能入口第三，标题和装饰第四。
```

### 3. 首页设置弹窗

```text
首页上的设置弹窗。背景仍能隐约看到首页草地场景，但被深色半透明蒙层压暗。中央是一张云朵白圆角弹窗卡片，描边为深墨绿色手绘线条，顶部可以有卡皮巴拉探头或小花装饰。标题居中写“设置”。右上角放关闭按钮，但按钮位于弹窗内部，不要贴到全屏右上角。中部是三行设置项：音效、震动、音乐，每一行采用“左侧小图标 + 标签文字 + 右侧开关”的布局，条目做成柔软的浅色圆角底板，不使用硬分割线。底部单独放一个“问题反馈”入口按钮，位置稳定、尺寸适中。整体感觉要轻量、清楚、好关闭，不要做成厚重后台面板。
```

### 4. 圈子好礼弹窗

```text
首页功能弹窗“圈子好礼”。半透明深色蒙层之上，居中弹出米白色圆角奖励卡片。顶部标题是“圈子好礼”，下方一行简短副标题说明“加入游戏圈可领取奖励”。中部奖励展示区是视觉中心，重点突出“100 金币”，可以用大金币堆、闪亮星星、小花和彩带营造轻庆祝感，但仍然保持儿童绘本的柔和气质。主操作区域需要预留明显而宽敞的微信原生按钮位置，周围不要被装饰元素挤压。底部可放一句轻量说明“完成跳转后返回游戏即可领取”。关闭按钮可以放在右上角或底部次级位置，但不能抢主按钮视觉。整体体验是轻松领一份好礼，而不是功能教程。
```

### 5. 添加桌面弹窗

```text
首页功能弹窗“添加桌面”。中央居中弹窗采用米白色圆角卡片，深墨绿色手绘描边。头部标题写“添加桌面”，副标题说明“添加后每日可领奖励”。中部示意区用一张非常容易看懂的手绘小插图来表现“小游戏图标进入手机桌面”的过程，构图简单清楚，可以有手机、箭头、桌面图标、卡皮巴拉小助手。示意区下方是奖励区，突出“今日首次奖励 50 金币”。底部是双按钮布局，主按钮“去添加”使用暖橙色厚圆角 CTA，次按钮“稍后再说”弱化处理。整体顺序必须清晰：标题与副标题、示意图、奖励、主按钮、次按钮。
```

### 6. 每日奖励弹窗

```text
首页功能弹窗“每日奖励”或“今日签到”。中央为米白色圆角签到弹窗，顶部有欢呼的卡皮巴拉和星星、小花装饰。标题区写“每日奖励”，并显示“已连续签到 X 天”的次级信息。弹窗中部是 7 格签到日历卡片区，每格展示第几天和对应金币数，卡片风格统一圆润可爱。状态必须非常清楚：已领取的卡片有完成勾勾与浅色完成态；今日可领取的卡片明显放大、高亮、描边更强，还带轻微发光或奖励感；未来未解锁的卡片则柔和灰化。日历下方单独放大展示“今日奖励”，让玩家一眼看到今天能拿多少。底部操作区如果今天可领取，就放一个醒目的“领取奖励”主按钮；如果今天已领取，则显示完成态按钮或完成提示。整体中心必须是“今天这格”和“今日奖励”。
```

### 7. 商店页 - 形象 Tab

```text
微信小游戏商店页，全屏功能页，激活的是“形象”标签。背景延续同一草地世界，但亮度和细节比首页更弱，避免干扰阅读。顶部栏统一结构：左侧返回按钮，中间标题“商店”，右侧或右上安全区域左侧放金币余额显示胶囊，整体避开微信胶囊按钮。顶部栏下方是一组横向页签“形象 / 道具 / 音效”，当前“形象”标签高亮。内容区使用大卡片纵向列表，每张商品卡都包含形象预览图、名称、价格、状态标签、购买按钮。已拥有的卡片显示“已拥有”，当前使用中的卡片显示“当前使用中”，未开放的卡片显示“敬请期待”。按钮位置在所有卡片内保持统一，卡片不要做得太小。整体是温暖、整洁、有收藏感的功能页，而不是廉价堆满商品的商城。
```

### 8. 商店页 - 道具 Tab

```text

```

### 9. 商店页 - 音效 Tab


### 10. 主题页

```text
微信小游戏主题页，全屏竖屏功能页，整体仍属于同一个儿童绘本世界。顶部栏与商店页统一：左侧返回按钮，中间标题“主题”，顶部布局避开微信胶囊按钮。页面明显分为上下两部分。上半部分是大幅主题预览区，展示当前选中主题的背景预览，并叠加 2 到 4 个主题图块样例，帮助玩家快速理解主题差异。中部信息区展示主题名称、一句简短气质说明，以及解锁条件或已解锁状态。下半部分是主题卡片纵向列表，每张卡片包含缩略图、名称、状态标签。底部操作区根据状态显示“使用该主题”“当前使用中”或“通关第 X 关解锁”这类清晰文案。默认主题、已解锁主题、当前使用主题、未解锁主题四种状态必须一眼可分。整体像一个温柔的收藏陈列页，同时具备明确选择动作。
```

### 12. 排行榜页 - 好友榜

### 13. 局内页面 - 通用布局

```text
微信小游戏局内主界面，竖屏，整体遵循“棋盘优先”。背景使用当前主题背景，但视觉焦点必须避开棋盘后方中心区域，让中轴保持低对比、低纹理、低饱和，方便读盘。顶部是横向扁平 HUD 区，包含菜单按钮、机制提示图标、关卡标题、进度条和宝箱，模块之间通过轻量小卡片分组，整体高度克制，避开顶部安全区与微信胶囊按钮。进度条是顶部最醒目的组件，宝箱是进度节点焦点。中部棋盘区必须严格水平居中，占据页面最大视觉面积，周围留足连线路径特效、消除特效和掉落动效空间。底部工具区是一块稳定的横向底板，三个工具位等距排列，每个工具位都有图标、库存数和点击反馈，缺货状态下整体灰化但仍可点击。底部工具区必须高于手势安全线，不要再叠加复杂信息。整体信息层级：棋盘第一，顶部进度第二，工具区第三，背景装饰最后。
```

### 14. 局内页面 - 水果主题


### 16. 局内页面 - 森林音乐会主题

```text
在“局内页面通用布局”基础上，把背景主题改为森林音乐会主题。场景像午后林间小舞台，顶部是透光树冠和悬挂小灯串，左右边缘布置木琴、铃鼓、三角铁、小鸟、音符和彩带，底部做成木质舞台边缘或草地舞台来承托工具区。主色以木色、苔绿、浅金、湖蓝为主，像阳光穿过树叶形成的温暖斑驳。整体要有轻快、有陪伴感的小型音乐会氛围，但绝不能做成强追光、强炫光、演唱会舞台那种刺眼风格。
```

### 17. 局内页面 - 植物主题

```text
在“局内页面通用布局”基础上，把背景主题改为植物主题。整体是会呼吸的森林或温室角落，顶部可以看到大片树叶、透光天窗和轻柔天光，左右边缘布置龟背竹、蕨类、蘑菇、垂落藤蔓，底部用湿润石板路、苔藓地面或木质花架来承托工具区。主色为森林绿、浅青绿、露珠白，点缀少量粉紫小花。边缘加入叶片投影、漂浮花粉、露珠高光，但笔触必须柔和。整体安静、清新、有氧气感，让人想深呼吸，不要变成浓重暗林。
```

### 18. 局内页面 - 动物主题

```text
在“局内页面通用布局”基础上，把背景主题改为动物主题。场景像河岸草坡上的动物朋友聚会，顶部是开阔天空和树枝，左右边缘安排小鸟停枝、乌龟趴石、猴子挂树、远处河边小动物等陪伴式角色，底部用浅草坡、河岸石块或木栈道承托工具区。主色为草绿、天蓝、暖黄、浅棕，整体比水果主题更自然，比植物主题更活泼。所有动物都必须待在边缘讲故事，不能围住棋盘。核心氛围是“水豚和万物交朋友”的轻喜剧陪伴感。
```

### 19. 机制说明蒙层

```text
局内开局时出现的机制说明蒙层。背景是当前局内页面，但被半透明深色遮罩轻轻压暗，不要完全遮死。中央是一张居中的米白色说明卡片，顶部有认真教学表情的卡皮巴拉。卡片中心先展示一个大机制图标，再放机制标题，例如“迷雾关卡”或“消除后向左贴合”，下面只用 1 到 2 句非常短的解释文案。底部可以有“点击任意位置继续”的轻量提示。整体要扫一眼就懂，避免长段说明，重点是高效、友好、可快速关闭。
```

### 20. 局内菜单设置弹窗

```text
局内暂停菜单弹窗。半透明深色蒙层压暗当前游戏界面，中央弹出一张米白色圆角菜单卡片。顶部标题写“菜单”或“设置”。中部先放三行开关项：音效、震动、音乐，结构与首页设置相同，但更紧凑直接。下方是动作按钮区，需要明确主次关系：最显眼的是“继续游戏”，其次是“重玩本关”，再次是“放弃挑战”，最底部可以弱化放“问题反馈”。危险操作“放弃挑战”不要使用最高饱和主按钮色。整体感觉要比首页设置更像一个暂停决策面板，操作清晰，不误触。
```

### 21. 放弃确认弹窗

```text
危险操作确认弹窗“放弃挑战”。中央弹窗顶部是带一点劝留情绪的卡皮巴拉插画，表情不舍但不沉重。中部优先展示当前关卡号和当前进度条，例如“已完成 72%”，让玩家先看到自己已经推进到哪里。进度信息下方放一句鼓励文案，例如“再努力一下就快过关啦”。底部是双按钮决策区，主按钮“再试试”必须做成最强视觉按钮，次按钮“放弃挑战并返回首页”明显弱化处理，不要高亮危险操作。整体目标是鼓励继续挑战，而不是只是完成确认流程。
```

### 22. 道具不足弹窗

```text
局内弹窗“道具不足”。中央米白色圆角卡片顶部有抱着空空道具袋、略显委屈的卡皮巴拉。标题写“道具不足”。中部展示当前缺少的具体道具图标、名称和一句简短功能说明，帮助玩家快速理解这个道具为什么重要。说明区下方单独做一个醒目的次数模块，显示今日剩余补给次数，例如“1/3”，数字要清楚，不要埋在段落里。底部主操作区放一个明确收益的按钮，例如“分享获得 1 个提示”或“观看视频获得 1 个刷新”。如果次数已经用尽，则按钮置灰并显示“今日已无补给次数”。整体重点是“缺什么”和“补给后得到什么”。
```

### 23. 主题解锁弹窗

```text
奖励感很强的主题解锁弹窗。背景是轻微压暗的当前页面，中间是一张比普通弹窗更高、更有仪式感的奖励卡片。顶部使用柔和光芒、粒子、主题徽章、花朵和彩带，并配一只非常开心的卡皮巴拉来制造庆祝感。中部是大尺寸主题展示区，清楚展示新主题名称、主题预览图以及一组代表性图块样式，让“新主题本身”成为第一视觉焦点。其下放一句简短情绪文案，例如“恭喜获得森林音乐会主题”。底部按钮区是双按钮：主按钮“立即使用”，次按钮“好的”。整体应比普通系统弹窗更有礼物感，但仍然保持儿童绘本的温暖柔和。
```

### 24. 每日挑战完成提示

```text
每日挑战完成后的轻量奖励提示弹窗。整体比主题解锁弹窗更轻，不要太厚重。中央弹窗标题写“今日挑战完成”。中部重点突出奖励数值“50 金币”，搭配少量金币、星星、花朵和卡皮巴拉鼓掌动作，营造轻盈庆祝感。旁边或下方附带“今日已完成”的状态标签，避免玩家重复进入产生疑问。底部只保留一个居中的单按钮“知道了”。整体节奏要轻快、明快，方便玩家快速确认后返回首页。
```

## 使用建议

- 如果你要先跑一轮统一风格图，建议优先生成：`首页`、`加载页`、`局内页面-水果主题`、`商店页-形象 Tab`、`主题页`、`排行榜页-世界榜`。
- 如果你要补弹窗体系，建议再依次生成：`首页设置弹窗`、`每日奖励弹窗`、`放弃确认弹窗`、`主题解锁弹窗`。
- 如果某个模型容易把 UI 画成插画海报，可以在模版末尾额外补一句：`This is a game UI layout concept, not a poster.` 或 `这是界面设计图，不是海报。`
