## 全局设计语言

**风格定义**：春日草地彩铅水彩儿童绘本风，整体色调与用户提供的首页参考图完全一致。画面必须保持高明度、低攻击性、轻空气感，核心视觉关系固定为“浅青蓝天空 + 黄绿色草坡 + 暖棕色卡皮巴拉 + 清晰黑色描边 + 暖橙 CTA”。

**参考图色调锚点**：
- 天空固定为浅青蓝渐变：`#BEEFFF` -> `#DDF7FF`
- 草地固定为黄绿色大底：`#CBEA72`，阴影草色：`#A8D95C`
- 主按钮固定为暖珊瑚橙：`#F28C63`，高光：`#FFD7B5`
- 描边固定为黑色：`#000000`
- 卡片固定为暖白：`#FFF9EF`
- 角色主体固定为暖棕色：`#BA7A47`
- 正文与标题文字固定为深绿字色，统一加黑色 2px 描边和轻阴影

**色彩体系**：
- 主背景色：黄绿色 `#CBEA72`
- 草地阴影色：浅草绿 `#A8D95C`
- 天空色：清透浅青蓝 `#BEEFFF`
- 主按钮色：暖珊瑚橙 `#F28C63`
- 奖励强调色：蒲公英黄 `#F6D55C`
- 描边色：黑色 `#000000`
- 弹窗/卡片底色：暖白 `#FFF9EF`
- 危险/提醒：暖红 `#E46A5D`
- 角色主体：暖棕色 `#BA7A47`

**标题文案规则**：
- 凡出现游戏品牌主标题、首页主标题、加载品牌标题、宣传主标题时，统一使用“开心点连连看”
- 不再使用其他游戏名或近义替代名
- 页面功能标题继续按页面语义书写，例如“设置”“商店”“主题”“排行榜”

**布局原则**：
- 全屏页面：加载页、首页、商店、主题、排行榜等，优先保证沉浸感和上下裁切余量。
- 局内页面：以棋盘为绝对主角，顶部 HUD 横向压缩，底部工具区稳住，不抢中部注意力。
- 弹窗页面：统一采用 `蒙层 + 主卡片 + 标题区 + 内容区 + 按钮区` 结构，主次动作明确。
- 信息层级：开始游戏 / 领取奖励 / 使用主题这类主按钮必须成为第一视觉焦点，其次才是数值和辅助功能。
- 适配原则：所有顶部元素避让安全区与胶囊按钮，所有底部交互避让手势安全线，所有文字统一使用黑色描边加阴影，且描边边界清晰，不再额外切换文字方案。

**角色与装饰原则**：
- 卡皮巴拉是情绪锚点，固定承担“陪伴、鼓励、庆祝、解释”的功能。
- 装饰元素固定使用蒲公英、白色小花、小鸭、蝴蝶、小木牌、草坡和云朵。
- 装饰永远服务于信息，不应包围主按钮或挤压点击区。

**AI 生图描述（加载页）**：

# 首页

**AI 生图描述（首页）**：  
全屏竖屏首页，春日草地彩铅水彩插画风，背景为固定治愈草地场景，不随主题切换。画面上方是干净天空区域，远景树影和淡云留白，方便承载顶部状态信息；中部偏下是浅绿草坡、蒲公英、小花和柔软的草地纹理。

# 局内页面

**AI 生图描述（局内页面）**：  
竖屏主游戏界面，整体遵循“棋盘优先”。背景使用当前主题背景图，但视觉焦点避开棋盘区域。顶部是扁平横向 HUD 区，放菜单按钮、机制提示图标、关卡标题、进度条和宝箱，整体为横向一字带布局，模块之间统一通过轻卡片分组，不要占用太多纵向空间。进度条是顶部最醒目的功能组件，宝箱作为进度节点视觉焦点。  
中部棋盘区为页面绝对主角，严格水平居中，占据最大视觉面积，周边留足连线路径、消除特效和掉落动效的安全边界。棋盘与顶部 HUD 之间保持清晰间距，任何额外装饰都不能压到棋盘上。  
底部工具区是一块稳定的横向底板，三个工具位等距排布，每个工具位包括图标、库存数和点击反馈。库存为 0 时工具位整体灰化，但仍保留可点击提示能力。工具位必须高于底部手势安全线，并且不要再叠加其他复杂信息。整体页面第一焦点永远是棋盘，第二是顶部进度，第三才是工具与辅助信息。

**局内主题背景补充（按 5 个主题分别出图）**：  
所有主题背景统一遵循“上轻下稳、中间净空”的构图模板。顶部固定放可裁切弱信息元素，左右两侧承担主题识别内容；棋盘后方中轴区域必须低对比、低纹理、低饱和；底部固定用承托面承接工具区，让下半部视觉更稳。

1. **水果主题背景图**：  
晴天果园与野餐场景，浅蓝天空、软云、垂落枝叶，左右边缘点缀苹果树枝、橘子树枝、半透明果片和轻盈果汁气泡，底部固定为浅木桌面。主色为蜜桃橙、草莓红、青柠绿、奶油白，整体清爽明亮，像夏日午后的果园野餐，但不要让完整大水果靠近棋盘边缘，默认主题应是五张里可读性最高、最稳定的一张。

2. **情绪主题背景图**：  
柔和的室内角落场景，顶部固定为渐变天幕，左右两侧像情绪贴纸墙，放低对比的表情贴纸、便利贴、手绘小图标，底部固定用抱枕和小舞台地板承托工具区。主色为雾蓝、淡紫、奶灰粉、暖米色，局部点少量橘红表达“冒火”“崩溃”等情绪，但必须克制。整体感觉是“内心有很多小剧场，但世界依然温柔”，避免阴暗压抑。

3. **森林音乐会主题背景图**：  
午后林间小舞台场景，顶部是透光树冠和悬挂小灯串，左右边缘布置木琴、铃鼓、三角铁、站在枝头的小鸟、零散音符和彩带，底部固定做成木质舞台边缘。主色为木色、苔绿、浅金、湖蓝，光感像阳光穿过叶缝形成的温暖斑驳。整体需要有轻快节奏和朋友聚会的陪伴感，但不能做成强追光、强演唱会舞台那种刺眼效果。

4. **植物主题背景图**：  
呼吸森林场景，顶部可见大片树叶、透光天窗和柔和天光，左右布置龟背竹、蕨类、蘑菇、垂落藤蔓，底部固定为湿润石板路。主色以森林绿 `#3D9B6E` 为核心，搭配浅青绿、露珠白和少量花朵粉紫。边缘固定加入叶片投影、漂浮花粉和露珠高光，整体安静、清新、有氧气感，像空气在缓慢呼吸，不要做成阴暗密林。

5. **动物主题背景图**：  
河岸草坡上的动物朋友聚会，顶部是开阔天空和树枝，左右边缘固定安排小鸟停枝、乌龟趴石、猴子挂树三种动物朋友，底部固定用浅草坡和河岸石块承托工具区。主色为草绿、天蓝、暖黄、浅棕，比水果主题更自然，比植物主题更活泼。动物要以“陪伴式边缘出现”为原则，不可一群动物围住棋盘，整体强调“水豚和万物都是朋友”的轻喜剧陪伴感。

## 通用输出要求

- 所有全屏页都应保留长屏设备上下裁切余量，关键元素集中在中部安全区域。
- 所有弹窗内部元素必须严格相对弹窗排版，不能出现像全屏坐标拼贴的感觉。
- 所有按钮都应大、圆、厚、可点击，主按钮和次按钮必须有清晰层级。
- 所有叠加在复杂背景上的文字，都统一使用黑色描边和轻阴影提升可读性，边界必须清晰，不要发糊。
- 所有描边元素、卡片边框和按钮轮廓都要清楚利落，避免模糊发虚的边缘表现。
- 首页、商店、主题、排行榜的顶部结构应尽量统一，减少页面切换认知成本。
- 局内页面的一切辅助信息都必须为棋盘让路。
- 所有页面默认以参考图那套浅天蓝、黄绿色、暖橙、暖白、深绿的色彩关系出图，不允许切换到更灰、更暗或更冷的方案。
- 文档中的控件位置、按钮文案、状态表现和装饰选择都按本文固定描述执行，不再自行做二选一扩展。
- 凡出现游戏品牌主标题，统一写为“开心点连连看”。
