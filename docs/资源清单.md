## 四、弹窗与浮层（§7～§10、§15～§20）

### 4.1 首页设置（§7）

- 弹窗标题条「设置」、关闭、三行开关（音效/震动/音乐）左侧小图标（可与全局开关复用）
- 「问题反馈」按钮底图

### 4.2 圈子好礼（§8）

- 标题区装饰（可选）
- **100 金币** 大图标展示
- 关闭（次级）

### 4.3 添加桌面（§9）

- 「小游戏图标 → 手机桌面」**示意插图**（简洁一条主视觉）
- 「去添加」「稍后再说」主次按钮

### 4.4 每日奖励（§10）

- **7 格签到日历**卡片：已领 / 今日可领 / 未解锁 **三种状态** 底板
- 今日格 **高亮**（放大框、描边、光效至少两种）
- 第 7 格可加 **循环** 小标识
- 「领取奖励」主按钮；已领完成态

### 4.5 机制说明蒙层（§15）

- 棋盘区上半透明压暗（程序或一张蒙层）
- **机制说明卡**：机制 **图标** + 标题条（每关机制不同可多套图标）

### 4.6 局内菜单（§16）

- 标题「菜单」或「设置」
- 音效/震动/音乐（同首页）
- **重玩本关**（主）、**放弃挑战**（次）、**继续游戏**、**问题反馈**

### 4.7 放弃确认（§17）

- **轻劝留插画**或卡皮巴拉 **表情** 小图
- 当前关卡号、进度条样式可与 HUD 统一组件

### 4.8 道具不足（§18）

- 标题「道具不足」
- **道具图标**（与局内三工具对应，可放大展示）
- **分享补给 / 激励视频** 按钮（主）

### 4.9 主题解锁（§19）

- **仪式感头部**：光芒、粒子、**主题徽章** 或装饰（可拆层叠加）
- 主题 **预览图**、**代表图块** 样式小图
- 「立即使用」「好的」

### 4.10 每日挑战完成（§20）

- 「今日挑战完成」标题区
- **50 金币** 突出展示
- 「今日已完成」**状态标签**
- 「知道了」单按钮

## 五、功能页：商店 / 主题 / 排行榜（§11～§13）

### 5.1 商店（§11）

| 类型 | 资源说明 |
|------|----------|
| 背景 | 延续首页气质、略弱细节的全屏或渐变底 |
| 顶栏 | 返回、标题「商店」、右侧金币 |
| 页签 | **形象 / 道具 / 音效** 三枚（选中/未选中） |
| 形象页 | 纵向大卡片：预览图、**已拥有 / 敬请期待 / 使用中** 标签、购买按钮 |
| 道具页 | 两列网格卡：道具 **图标**、名称、价格、购买 |
| 音效页 | 列表卡：封面 **图标**、风格描述、购买/已拥有、试听区占位 UI |

### 5.2 主题（§12）

| 类型 | 资源说明 |
|------|----------|
| 顶栏 | 返回、标题「主题」 |
| 预览区 | 当前选中主题 **大图预览**；可叠加 2～4 块 **样例图块** |
| 列表 | 每主题 **缩略图** + 名称 + 四态：**默认/已解锁/当前使用/未解锁**（未解锁柔和遮罩） |
| 底部 | 「使用该主题」「当前使用中」、解锁条件 **短标签** 样式 |

### 5.3 排行榜（§13）

| 类型 | 资源说明 |
|------|----------|
| 顶栏 | 返回、标题「排行榜」 |
| 页签 | **世界榜 / 好友榜** |
| 榜单卡 | 大卡片底板；每行：排名区、**头像占位**、昵称、关卡、时间（程序字+底板） |
| 前三 | 可选 **奖牌** 小图标（勿过大） |
| 好友榜 | 主域 **固定外框**；列表内容由开放数据域绘制，需 **固定窗口尺寸** 设计稿 |
| 空态 | **空榜占位图** + 文案 |

### 5.4 隐私协议（§13.4）

- 好友榜首次：**隐私确认弹窗**（与通用弹窗共用底板即可）

## 六、局内页（§14）

| 类型 | 资源说明 |
|------|----------|
| 背景 | **五套主题** 局内背景（见下「主题背景」），共用同一布局骨架 |
| HUD | **关卡标题**条、**进度条**、**宝箱**（含节点动效用序列帧或单图） |
| HUD | **机制提示** 小图标、**菜单** 按钮 |
| 棋盘 | 格子/图块若在 PIXI 绘制则归程序；若美术出 **图块皮肤** 需单独清单（与玩法一致） |
| Combo | 卡皮巴拉 **Combo 演出** 位置插画或序列帧（棋盘上沿/侧边） |
| 底部 | **三个工具位**：各工具 **图标** + 数字 + **缺货灰态**；**统一工具栏底板** |

### 6.1 五主题局内背景（§14.2.1，各 1 张或可滚动延展）

1. **水果**：果园/野餐/果汁气泡，明快清爽  
2. **情绪**：室内角落/梦境舞台、贴纸式边缘装饰  
3. **森林音乐会**：林间舞台、音符飘带（不穿过棋盘中心净空）  
4. **植物**：温室/森林、苔绿主色、露珠叶影  
5. **动物**：河岸/草坡、动物边缘探头  

> 构图统一：**上轻下稳、中间净空**；左右边缘叙事，棋盘后方中轴低对比。

### 6.2 首页与局外的「主题」关系（§6 vs §12）

- 设计文档写明：**首页背景固定**，不随主题切换；**局内**使用当前主题背景。若产品后续允许首页随主题变，再增首页背景变体。

## 七、音频资源（依据 `docs/音效设计.md` + 局内体验惯例）

### 7.1 界面与反馈（Organic / Crispy）

| 用途 | 说明 | 音效设计文档对应 |
|------|------|------------------|
| 通用点击 / 选中 | 短促轻盈 | 泡泡破裂、木质敲击、Soft UI Tick |
| 弹窗打开 / 关闭 | 轻柔一层（可选，避免刺耳） | 与 UI Tick 同系或更轻 |

### 7.2 局内核心

| 用途 | 说明 | 音效设计文档对应 |
|------|------|------------------|
| 成功消除 / Match | 满足感 | 风铃、魔法闪烁、水滴/水泡 |
| 连击 / Combo | 可阶梯升调 | 琶音、高频叮当、短奖励乐句 |
| 星星飞向进度 / 收集 | 配合吸入动画 | 柔和 Whoosh、闪烁拖尾、收集 Ding |
| 进度满 / 宝箱 | 若与星星分轨 | 文档示例：success_bell、飞入宝箱 |

### 7.3 音乐（BGM）

| 用途 | 说明 |
|------|------|
| 首页 / 大厅 | 治愈、轻松循环，不抢戏 |
| 局内 | 可与大厅分轨，节奏略轻快或更淡（按产品选） |
| 商店 / 主题等 | 可选与首页同曲或更弱循环，减少切歌成本 |

> 文档建议来源：Casual / Cozy / Acoustic 类免版税包；注意微信内使用 **wx.createInnerAudioContext**，文件落地 **USER_DATA_PATH**（见 `AGENTS.md`）。

### 7.4 可选补充（设计文档未单列、实现常需）

| 用途 | 说明 |
|------|------|
| 失败 / 放弃 | 短促不挫败的提示音（与胜利区分） |
| 获得奖励 / 签到 | 轻快 Coin 或短 Jingle |
| 主题解锁 | 比每日完成略隆重的短乐句或 Chime |
| 宝箱开启 | 若与「星星入箱」分开展示，单独一轨更干净 |

## 八、与程序/图集约定的说明

- **序列帧**：加载页角色、首页待机、宝箱、Combo、红点呼吸等需列明 **帧率与是否循环**。  
- **九宫格**：弹窗底板、按钮、进度条槽建议标注可拉伸区域。  
- **开放数据域**：好友榜仅主域外框+固定区域需切图，内层由数据域绘制。  
- **棋盘图块**：若使用 TexturePacker 图集，在「图块皮肤」子清单中列 sprite 名与主题对应关系。
